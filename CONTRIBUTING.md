# Contributing to SEAD Framework

Thank you for your interest in contributing to the SEAD Framework!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest improvements
- Check existing issues before creating a new one
- Include clear reproduction steps for bugs
- For template improvements, explain the problem the change solves

### Suggesting Enhancements

- Open an issue with the "enhancement" label
- Describe the use case and expected benefit
- Reference the specific phase/template affected

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/improvement-name`)
3. Make your changes
4. Test your changes with a real project if possible
5. Commit with clear messages (`git commit -m "Add: description"`)
6. Push to your fork (`git push origin feature/improvement-name`)
7. Open a Pull Request

### Pull Request Guidelines

- Keep changes focused and atomic
- Update relevant documentation
- Add yourself to CONTRIBUTORS.md if this is your first contribution
- Reference any related issues

## Template Contribution Guidelines

When modifying templates:

- Maintain the existing structure unless proposing a structural change
- Keep templates tool-agnostic in the main files
- Tool-specific content goes in `docs/integrations/`
- Include examples where helpful
- Test templates in real projects before submitting

## Code of Conduct

- Be respectful and constructive
- Focus on the work, not the person
- Welcome newcomers
- Assume good intentions

## Questions?

Open a Discussion on GitHub for general questions about the framework.
