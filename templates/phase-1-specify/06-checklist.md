# Phase 1: SPECIFY - Checklist

Use this checklist to verify Phase 1 is complete before moving to Phase 2.

---

## Discovery

- [ ] Client/stakeholder interviews completed
- [ ] All input documents collected
- [ ] User types identified
- [ ] Compliance requirements documented
- [ ] Timeline confirmed

---

## PRD

- [ ] Executive summary written
- [ ] Problem statement with pain points documented
- [ ] Success metrics defined (quantifiable)
- [ ] User types and permissions defined
- [ ] All features listed with IDs
- [ ] Feature descriptions are clear and specific
- [ ] Acceptance criteria defined for each feature
- [ ] AI features documented (if applicable)
  - [ ] AI capabilities listed
  - [ ] AI boundaries defined
  - [ ] Human confirmation points identified
- [ ] Integration requirements documented
- [ ] Non-functional requirements specified
  - [ ] Performance requirements
  - [ ] Security requirements
  - [ ] Scalability requirements
- [ ] Compliance requirements identified
- [ ] Out of scope items listed
- [ ] Open questions documented
- [ ] Timeline/phases defined
- [ ] **PRD reviewed by stakeholders**
- [ ] **PRD approved**

---

## Feature-Test Matrix

- [ ] All PRD features included
- [ ] Each feature has acceptance criteria (Given/When/Then)
- [ ] Test scenarios cover:
  - [ ] Happy path
  - [ ] Validation errors
  - [ ] Edge cases
  - [ ] Error states
- [ ] AI features have boundary tests
- [ ] Integration features have failure scenarios
- [ ] Priority assigned (P0/P1/P2)
- [ ] Risk classification assigned (Critical/High/Medium/Low)
- [ ] Size assigned (S/M/L/Epic)
- [ ] **No L or Epic without decomposition plan**
